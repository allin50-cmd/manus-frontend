## Cross-Cutting Findings

### Finding 1 — Build is broken across all P1 systems

`npm run build` fails with `error TS2688: Cannot find type definition file for 'vite/client'`. This is a single shared build failure that blocks all three P1 systems in this repo.

### Finding 2 — Tests cannot run

`vitest` is listed in `devDependencies` but is not installed in `node_modules`. `npm test` fails immediately.

### Finding 3 — Two database schemas, one connection string

`server/db/schema.ts` (brand-suite: leads, intakeForms, complianceBundles, monitoredCompanies, contacts, deploymentStatus) and `server/drizzle/schema.ts` (ClerkOS: tenants, users, cases, hearings, documents, bundles, allocations, diaries, auditEvents) both use `DATABASE_URL`. The `drizzle.config.ts` only points at `server/db/schema.ts`. The ClerkOS schema has no managed migration path.

### Finding 4 — Two competing CI/CD pipelines

`azure-static-web-apps-ci-cd.yml` uses `npm ci` and targets Azure Static Web Apps. `deploy-vaultline.yml` uses `pnpm install --frozen-lockfile` and targets Azure App Service. Both trigger on push to `main`. The repo has a `package-lock.json` (npm) but `package.json` specifies `"pnpm": ">=8.0.0"` in engines. This is contradictory.

### Finding 5 — No email provider

No email delivery service (SendGrid, Mailgun, Resend, AWS SES, Nodemailer, SMTP) is imported or configured in any server file. FineGuard's core promise ("alerts delivered") has no technical implementation.

### Finding 6 — Hosting policy not implemented

The portfolio hosting policy specifies Vercel + Cloudflare. No `vercel.json` and no `wrangler.toml` exist in this repository. All CI/CD pipelines target Azure. This repo is not aligned to the hosting policy.

### Finding 7 — Azure SQL vs PostgreSQL mismatch

`deploy/main.bicep` provisions `Microsoft.Sql/servers` (Azure SQL Server / MSSQL). The application stack uses the `postgres` npm driver. These are incompatible. Either the Bicep template is wrong, the application driver is wrong, or a separate PostgreSQL resource was provisioned outside the IaC and is not documented.

### Finding 8 — No FineGuard alert scheduler

The `CompaniesHouseService` class exists and is functional for on-demand lookups. However, there is no scheduled job, cron, Azure Function timer trigger, or background worker that periodically polls Companies House for changes to monitored companies and triggers alerts. The Stripe webhook activates monitoring but nothing subsequently checks for compliance changes.

### Finding 9 — Marketing pages not routed

`src/pages/FineGuard.tsx`, `src/pages/VaultLine.tsx`, `src/pages/UltAi.tsx`, `src/pages/About.tsx`, `src/pages/BookDemo.tsx`, `src/pages/Pricing.tsx` exist but are not included in `src/App.tsx` routes. The deployed frontend only serves the ClerkOS operational dashboard (Cases, Hearings, Documents, Queue, Diary, Bundles).
