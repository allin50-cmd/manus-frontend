# Read This First

This is a real working product, not theory.

Core rule:

One company workspace. Many enabled app modules. No app silos.

Users log into their business workspace, not separate products.

Company is always the hub.

Applications are capabilities inside the workspace:

- FineGuard
- ClerkOS
- SmartReceptionist
- AutoLawClerk
- MediaManager
- BusinessAnywhereOS

Before coding:

1. Check what already exists.
2. Reuse before rebuilding.
3. Keep UX simple.
4. Preserve the working production deployment.
5. Ship real customer workflows, not architecture theatre.
